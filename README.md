## Блок 1 — SQL анализ и валидация данных

### 1. Развёртывание окружения
Для выполнения задания я поднял полностью изолированное окружение с помощью **Docker Compose**.

**Что было развернуто:**
- **PostgreSQL 16** — основная база данных
- **Apache Airflow 2.11.1** (LocalExecutor) — для удобства работы с DAGами в будущем

**Ключевые настройки:**
- Postgres доступен на порту **`5433`** (чтобы не конфликтовал с другими проектами)
- Airflow Web UI доступен на порту **`8081`**
- Использован официальный образ Airflow с минимальными изменениями

**docker-compose.yml** был взят за основу из официальной документации Airflow и адаптирован под задание.



## Что сделано

1. Развёрнуто локальное окружение с помощью `docker-compose.yml`:
   - Запущен контейнер PostgreSQL (порт `5433` на хосте).
   - Запущен Airflow webserver (порт `8081` на хосте).

2. Подготовлены и импортированы данные:
   - `orders_raw.csv` и `orders_agg.csv` скопированы в контейнер Postgres и загружены в таблицы `orders_raw` и `orders_agg` (см. раздел «Импорт CSV»).

3. Проверки:
   - выполнены контрольные SQL-запросы по количеству строк и поиску несоответствий между `orders_raw` и `orders_agg`.
   - результаты проверены через GUI — `DBeaver`.

4. Примечания:
   - учтены возможные проблемы: права на файлы, кодировка CSV (рекомендуется UTF-8), формат даты/времени в `created_at`.
